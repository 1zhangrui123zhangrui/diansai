# 串口屏通信方案（最终版）

## 一、方案概述

采用"屏主动上报页码 + 单片机直接写当前页控件"方案。

## 二、屏端改动

### 2.1 统一控件命名

每个页面上显示坐标的文本控件，objname 必须统一：

| 页面 | X 坐标控件 objname | Y 坐标控件 objname |
|---|---|---|
| 首页 (page 0) | `tX` | `tY` |
| 启动页面 (page 1) | `tX` | `tY` |
| 区域巡逻 (page 2) | `tX` | `tY` |
| 顺序巡逻 (page 3) | `tX` | `tY` |
| 自动巡逻 (page 4) | `tX` | `tY` |
| 火源坐标显示 (page 5) | `tX` | `tY` |

不同页面可以有同名控件，屏不带页面前缀时只在当前页面查找。

### 2.2 火源坐标显示页的额外控件

在火源坐标显示页面（page 5）上额外放 4 个文本控件：

| 控件 | objname | 用途 | 初始值 |
|---|---|---|---|
| 火源1 X 坐标 | `tF1X` | 第1个火源的X | `--` |
| 火源1 Y 坐标 | `tF1Y` | 第1个火源的Y | `--` |
| 火源2 X 坐标 | `tF2X` | 第2个火源的X | `--` |
| 火源2 Y 坐标 | `tF2Y` | 第2个火源的Y | `--` |

### 2.3 每个页面的后初始化事件

每个页面的**后初始化事件**里加页码上报，告诉单片机当前在哪一页：

**首页：**
```
prints "UU",0
prints 0xB0,1
```

**启动页面：**
```
prints "UU",0
prints 0xB1,1
```

**区域巡逻：**
```
prints "UU",0
prints 0xB2,1
```

**顺序巡逻：**
```
prints "UU",0
prints 0xB3,1
```

**自动巡逻：**
```
prints "UU",0
prints 0xB4,1
```

**火源坐标显示：**
```
prints "UU",0
prints 0xB5,1
```

### 2.4 按钮事件

所有按钮的弹起事件里，**prints 必须放在 page 跳页之前**，否则跳页后 prints 可能不执行。

**启动按钮 b0（首页）：**
```
prints "UU",0
prints 0xA1,1
page 1
```

**区域巡逻按钮 b1（首页）：**
```
prints "UU",0
prints 0xA2,1
page 2
```

**顺序巡逻按钮 b2（首页）：**
```
prints "UU",0
prints 0xA3,1
page 3
```

**自动巡逻按钮 b3（首页）：**
```
prints "UU",0
prints 0xA4,1
page 4
```

**火源坐标按钮 b4（首页）：**
```
page 5
```

**返回首页按钮（各子页面）：**
```
prints "UU",0
prints 0xA0,1
page 0
```

**顺序巡逻确认按钮 b1（顺序巡逻页面）：**
```
prints "UU",0
prints 0xA3,1
prints t1.txt,0
prints 0xFE,1
t1.txt=""
```

## 三、通信协议

### 3.1 屏 → 单片机（命令码）

帧格式：`0x55 0x55 + CMD`

| 命令 | 码值 | 说明 |
|---|---|---|
| 返回/急停 | `0xA0` | 停止当前任务 |
| 启动 | `0xA1` | 回到中心区域 |
| 区域巡逻 | `0xA2` | 四角依次巡逻 |
| 顺序巡逻 | `0xA3` | 后跟 5 字节 ASCII + 0xFE |
| 自动巡逻 | `0xA4` | 蛇形巡逻 |
| 标定 | `0xA5` | 零点标定 |

### 3.2 屏 → 单片机（页码上报）

帧格式：`0x55 0x55 + 0xBx`

| 页码 | 码值 | 页面 |
|---|---|---|
| 0 | `0xB0` | 首页 |
| 1 | `0xB1` | 启动页面 |
| 2 | `0xB2` | 区域巡逻 |
| 3 | `0xB3` | 顺序巡逻 |
| 4 | `0xB4` | 自动巡逻 |
| 5 | `0xB5` | 火源坐标显示 |

### 3.3 单片机 → 屏（实时坐标）

ASCII 指令 + 3 个 0xFF 结束符，**不带页面前缀**：

```
tX.txt="12.34" FF FF FF
tY.txt="-5.67" FF FF FF
```

每 100ms 发一次，屏自动更新当前页面上叫 `tX` / `tY` 的控件。

### 3.4 单片机 → 屏（火源坐标）

检测到火源时发一次，或用户切到火源页面时补发：

```
tF1X.txt="20.15" FF FF FF
tF1Y.txt="-10.30" FF FF FF
tF2X.txt="5.50" FF FF FF
tF2Y.txt="18.20" FF FF FF
```

## 五、数据流总图

### 5.1 实时坐标

```
单片机主循环 每100ms
  ↓
Screen_SetCoord(x, y)
  ↓
发送 tX.txt="12.34" + FF FF FF  (不带页面前缀)
  ↓
屏在当前页面查找 tX 控件 → 立刻更新显示
```

### 5.2 屏按钮命令

```
用户按"启动"按钮
  ↓
屏发送 55 55 A1 (prints 在 page 之前执行)
  ↓
屏跳转到 page 1
  ↓
page 1 后初始化事件发送 55 55 B1 (页码上报)
  ↓
单片机收到 A1 → 触发 on_start 回调
单片机收到 B1 → 记录 current_page = 1
```

### 5.3 火源坐标

```
蛇形巡逻中,nRF 收到 fire_flag=1
  ↓
task.c: 停车 → 蜂鸣器响3声
  ↓
Screen_RecordFire(x, y)  // 存在单片机内存里
  ↓
如果当前在火源页 → 立刻发送 tF1X.txt="xx" 
如果当前不在      → 存着不发
  ↓
用户稍后切到火源页面
  ↓
屏发 55 55 B5 → 单片机收到 → Screen_FlushFire()
  ↓
发送 tF1X/tF1Y/tF2X/tF2Y → 屏显示
```

## 七、注意事项

1. **按钮事件里 prints 必须在 page 之前**，否则跳页后 prints 可能不执行
2. **每个页面的坐标文本控件必须叫 `tX` 和 `tY`**，名字不一致会导致赋值无效
3. **火源坐标存在单片机内存里**，最多 2 个，掉电丢失
4. **单片机发坐标不要太快**，100ms 一次（10Hz）足够，太快会撑爆屏的串口缓冲
5. **屏的波特率和单片机必须一致**，都是 115200
6. **不再需要全局变量控件、定时器、前初始化赋值**，可以全部删掉简化工程
